博客迁移到了HEXO

Author: 之行 Date: March 27, 2022  9:35:36

用了七年的域名ieza.cn今年开始没有再续费了，一来是因为博客的更新越来越少，很多文字性的东西已经很难再沉下心去写，二来也是开始尝试用视频记载一些生活中的片段，发布平台也都是在B站之类的视频网站。但是从14年做博客到现在，从折腾各种主题到自己拼凑了一款模板并且还有不少人留言索要，也总归是从14岁高一到如今快22研一的留念地和一些记忆的载体。

没有过多记述的，来把自己从typecho转Hexo的过程写下来分享下吧，网络上有一些插件形式的东西实际上过于复杂了。

这里只需要借助两个工具：

一个是typecho转wordpress的插件：https://github.com/sunxiyuan/ByeTyp

二是wordpress转hexo的插件：https://hexo.io/zh-cn/docs/migration

这样应该算是最简单的转换方式。

接下来就是Hexo在本地的安装和基于gitee的托管，由于我是mac m1系统，相比于网络上大量的windows环境下的安装步骤还是有所不同的，因此也贴下面。

首先安装nodejs以及git，不需要通过Homebrew之类的工具，mac直接安装是最便捷的。

nodejs: https://nodejs.org/en/download/

Git: https://sourceforge.net/projects/git-osx-installer/

Nodejs安装好后检查版本：

node -v

npm -v

mac初次安装git会提示安装一个工具，等待即可，完成后检查。

接着安装hexo：

npm install -g hexo-cli

但这里mac m1基本会提示权限错误，再次：

sudo npm install -g hexo-cli

安装后执行（创建目录）：

hexo init blog

cd到目录：

cd blog

安装：

npm install

启动：

hexo server

接下来就是Hexo部署到gitee，这里就很多帖子了，但是现在新版的gitee开启pages还需要手持身份证拍照。

hexo的主题和设置都需要直接在文本页操作，路径是在themes内主题下的_config.yml和layout

最后是上传到Github，Github创建仓库后在Hexo的config.yml下添加：

deploy:

type: git

repo: https://github.com/xxx/xxx.github.io.git

branch: main

需要注意的是github目前默认的branch是main。

接着在github的设置里添加令牌：Settings/Developer settings/Personal access tokens 勾选所有设置

回到终端，cd到hexo目录。

hexo clean

hexo g

hexo d

最后一部操作时会提示输入github账号密码，由于新版本添加了令牌也只能每次都复制粘贴，输入后正常上传即可。
