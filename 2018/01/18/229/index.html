机场没飞机日志

Author: 之行 Date: January 18, 2018  14:17:00 Category: 笔记

18/1/18更新

一有活动就剁手导致现在吃灰了那么几台机子 闲着无用 就搭建ss-panel共享 过程倒没那么顺利Orz  摘几个常用文档免得又到处翻

Links

 ss-panel Github  修改版(我也不知道改了啥反正先留着)  一个升级版前端  manyuser

Web documents 

Nginx配置 主目录在public下

root /www/ss-panel/public;
location / {
try_files $uri $uri/ /index.php$is_args$args;
}

开启目录下所需数据库(.env)

curl -sS https://getcomposer.org/installer php
php composer.phar install

字段配合后端

根目录下执行

php xcat createAdmin

创建管理员账户

manyuser配置

shadowsocks目录下 cp config_example.py为config.py进行配置

启用 MultiUser API

API_ENABLED = True

就是在你的站点地址后面加个 /mu

API_URL = ‘http://ss.prinzeugen.net/mu‘

还记得上面在 .env 中填写的 muKey 吗？把它填在这里

API_PASS = ‘api_key_just_for_test’

最后执行前端目录下python servers.py运行

Others

Ⅰ.端口开放

端口检查 执行

netstat -anp grep 你的端口

 端口开放及开放注册端口使用

 iptables -I INPUT -p tcp -m tcp –dport 你的端口 -j ACCEPT

允许你的端口及以上的端口  也可以指定 xxxxx:xxxxx 这样的范围

 iptables-save

Ⅱ.利用supervisord守护进程

1.centos安装并运行

yum install python-pip -y
pip install supervisor

2.输出配置并运行

echo_supervisord_conf > /etc/supervisord.conf
supervisord

3.配置文件 /etc/supervisord.conf

[program:ss-manyuser]
command = python /root/shadowsocks-py-mu/shadowsocks/servers.py
user = root
autostart = true
autorestart = true

4.重启并查看运行

killall -HUP supervisord
ps -ef grep servers.py
